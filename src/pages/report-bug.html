Reporting Tips

  • Be specific — include the exact URL or page where the bug appears.
  • Include clear steps to reproduce so maintainers can verify the issue.
  • Attach screenshots or screen recordings for visual bugs.
  • For security vulnerabilities, describe potential impact and attack vector.

Severity Guide

  • CriticalSystem compromise, data breach
  • HighSignificant data exposure
  • MediumLimited impact, partial access
  • LowMinimal impact on users
  • InfoBest practice suggestion
